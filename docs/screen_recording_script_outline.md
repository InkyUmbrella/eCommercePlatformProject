# 电商平台录屏脚本大纲（含页面路径与关键点击）

最后更新：2026-03-14
适用范围：`src/frontend/frontend-core` 当前实现路由

## 1. 录屏目标

用一条完整用户链路演示系统核心能力：

1. 注册与登录
2. 浏览商品与查看详情
3. 地址管理
4. 加购与结算下单
5. 订单查询与售后动作
6. 客服留言

## 2. 录屏前准备

1. 启动后端：`cd src/backend && python manage.py runserver`
2. 启动前端：`cd src/frontend/frontend-core && npm run dev`
3. 打开浏览器：`http://127.0.0.1:5173/`
4. 准备演示账号：一个普通用户（可现场注册）
5. 准备演示数据：至少 1 个可售商品、可下单库存
6. 可选：准备管理员 token（用于客服“回复模式”演示）

## 3. 页面路径总览

1. 注册页：`/`
2. 登录页：`/login`
3. 首页：`/home`
4. 商品列表：`/products`
5. 商品详情：`/product-detail/:productId`
6. 地址编辑：`/address-edit`
7. 购物车：`/cart`
8. 确认订单：`/checkout`
9. 下单成功：`/order-success`
10. 订单列表：`/orders`
11. 客服留言：`/support`

## 4. 录屏分镜脚本（建议 8-12 分钟）

### 4.1 开场与环境确认（0:00-0:30）

1. 展示项目名与打开地址 `http://127.0.0.1:5173/`
2. 说明“将从注册开始完整走通下单链路”

### 4.2 注册与登录（0:30-1:40）

页面路径：`/`
关键点击：

1. 输入昵称、密码、确认密码
2. 点击“立即注册”
3. 点击“去登录”

页面路径：`/login`
关键点击：

1. 输入用户名与密码
2. 点击“立即登录”

验收点：成功跳转到首页 `/home`

### 4.3 首页浏览与搜索（1:40-2:30）

页面路径：`/home`
关键点击：

1. 在搜索框输入关键词后点击“搜索”
2. 点击顶部导航“购物车/订单/客服”各一次（快速展示全局导航）
3. 点击“新品上架”任意商品卡片

验收点：成功进入商品详情 `/product-detail/:productId`

### 4.4 商品详情与地址管理（2:30-4:10）

页面路径：`/product-detail/:productId`
关键点击：

1. 点击数量“+/-”展示数量联动
2. 点击“修改”进入地址管理

页面路径：`/address-edit`
关键点击：

1. 填写“收货人/手机号/省市区/详细地址”
2. 勾选“设为默认收货地址”（建议）
3. 点击“保存”或“确认保存”

返回页面路径：`/product-detail/:productId`
关键点击：

1. 点击“加入购物车”
2. 点击“立即购买”（可选，直接跳结算）

验收点：出现加入购物车成功提示，或进入结算页 `/checkout`

### 4.5 购物车结算（4:10-5:20）

页面路径：`/cart`
关键点击：

1. 勾选商品复选框（含“全选”演示）
2. 调整数量“+/-”
3. 点击“去结算”

验收点：跳转 `/checkout`

### 4.6 提交订单与成功页（5:20-6:40）

页面路径：`/checkout`
关键点击：

1. 选择收货地址卡片
2. 选择支付方式（微信/支付宝/银行卡任一）
3. 点击“提交订单”

页面路径：`/order-success`
关键点击：

1. 展示订单号、金额
2. 点击“查看我的订单”

验收点：进入订单列表 `/orders`

### 4.7 订单列表操作（6:40-8:30）

页面路径：`/orders`
关键点击：

1. 点击标签切换：全部订单/待支付/待发货/待收货/已完成/售后中/已取消
2. 点击“搜索”并展示筛选结果
3. 根据订单状态演示按钮（按实际出现为准）：
   - “查看物流”
   - “取消订单”
   - “申请退款”
   - “确认收货”
   - “再次购买”

验收点：状态变化可见，列表刷新正常

### 4.8 客服留言（8:30-9:40）

页面路径：`/support`
关键点击：

1. 输入昵称、联系方式、留言内容
2. 点击“提交留言”
3. 点击“刷新”查看最新留言

可选（管理员演示）：

1. 输入管理员 token，点击“保存 Token”
2. 在某条留言输入回复，点击“提交回复”

验收点：留言新增成功；管理员模式下可回复

### 4.9 收尾（9:40-10:00）

1. 回到 `/home` 或 `/orders`
2. 总结“注册-下单-订单-客服链路已闭环”

## 5. 录屏话术模板（可直接照读）

1. “现在从注册页开始，先创建一个测试账号。”
2. “登录后进入首页，我先搜索商品并进入详情页。”
3. “这里可以修改收货地址，并把商品加入购物车。”
4. “接下来在购物车结算，提交订单后会进入成功页。”
5. “最后在订单列表查看状态流转，再演示客服留言能力。”

## 6. 录屏质量要求

1. 分辨率建议 `1920x1080`，帧率 `30fps`
2. 全程使用同一浏览器窗口，避免频繁切屏
3. 操作间隔保持 1-2 秒，关键数据停留可读
4. 出错不剪掉，保留“异常处理与恢复”更有说服力
5. 录屏文件建议命名：`ecommerce_demo_YYYYMMDD.mp4`
